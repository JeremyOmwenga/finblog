Automation: Awesome stuff

Jeremy Angwenyi 2 min read 255 words

If you are seeing this it means this Automation workflow has been successful.

Your opening paragraph goes here. This becomes the first thing readers see after the title. No special formatting needed — just write normally. A strong opening paragraph sets the tone for the whole piece.

This is your second paragraph. Keep writing. Each blank line between paragraphs creates a new <p> tag automatically.

This is a subheading

Use two hash symbols for section headings. They render as styled <h2> elements in the article template.

Everything under a heading is just normal text until the next heading or end of the article.

Smaller subheading (optional)

Three hashes give you a slightly smaller heading if you need one.

Using blockquotes

Put a > before a line to make it a styled pull quote:

This line will be styled as a highlighted blockquote with the red left border from your site's design.

Bold, italic, links

You can use bold text by wrapping words in double asterisks.

Use italic text with single asterisks. Or underscores work too.

Add a link like this: link text

Lists

Unordered list:

  • First item
  • Second item
  • Third item

Numbered list:

  1. First step
  2. Second step
  3. Third step

Code (optional)

Inline code uses backticks: like this.

A fenced code block uses triple backticks:

const example = "your code here";
console.log(example);

That's all you need. Save this file with a descriptive name like:

articles/nse-overheated-analysis.md

The filename becomes the URL of the article:

nse-overheated-analysis.html

Push it to GitHub and the rest happens automatically.
